# 薬局管理システム：管理者画面設計書

## 1. 概要

### 1.1 管理者画面の目的
プラットフォームの運営管理者が、システム全体の監視・管理・承認業務を行うための管理画面

### 1.2 主要な役割
- ユーザー管理（薬局・薬剤師）
- 資格証明書の確認・承認（重要）
- 請求書・支払い管理（重要）
- 契約管理
- ペナルティ管理
- システム設定

## 付録B：重要な運用フロー

### B.1 資格証明書承認フロー

\`\`\`
1. 薬剤師が証明書アップロード
   ↓
2. 管理者が「未確認」一覧で確認
   ↓
3. PDF表示で内容確認
   ├─ 免許証の確認
   └─ 登録票の確認
   ↓
4. 承認 or 差し戻し
   ├─ 承認 → 薬剤師に通知、応募可能に
   └─ 差し戻し → 理由入力、薬剤師に通知
\`\`\`

### B.2 支払い確認フロー

\`\`\`
1. 薬局が支払い報告
   ↓
2. 管理者が「確認待ち」一覧で確認
   ↓
3. 銀行口座で入金確認
   ├─ 金額の確認
   └─ 名義人の確認
   ↓
4. 確認完了 or 確認失敗
   ├─ 完了 → 個人情報開示、双方に通知
   └─ 失敗 → 理由入力、薬局に通知
\`\`\`

### B.3 ペナルティ解除フロー

\`\`\`
1. 薬局が解除申請
   ↓
2. 管理者が「申請中」一覧で確認
   ↓
3. 入金確認
   ↓
4. 解除承認 or 解除拒否
   ├─ 承認 → アカウント復旧、薬局に通知
   └─ 拒否 → 理由入力、薬局に通知
\`\`\`
