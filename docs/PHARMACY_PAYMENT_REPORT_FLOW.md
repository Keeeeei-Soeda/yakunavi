# 薬局の振込・支払い報告の一連の流れ

薬局が手数料を振り込んだ後、専用フォームで入金報告を行うときの流れをまとめています。

---

## 1. 前提（報告の直前まで）

- 契約が「手数料支払い待ち」（`pending_payment`）になっている
- 請求書が発行され、薬局は振込先・金額・期限を確認済み
- 薬局が銀行等で手数料を振り込んだ

---

## 2. 薬局側：支払い報告フォームへのアクセス

| 項目 | 内容 |
|------|------|
| **入口** | 薬局ダッシュボード → **請求書・支払い管理**（`/pharmacy/payments`） |
| **一覧** | 自薬局の請求書一覧が表示される。ステータスは「支払い待ち」「支払い報告済み」「支払い確認済み」など |
| **該当請求書** | 振り込んだ請求書の行の「詳細」等から **請求書詳細**（`/pharmacy/payments/[id]`）へ遷移 |

※ 契約詳細（`/pharmacy/contracts/[id]`）から「請求書を見る」で同じ詳細ページに飛べる場合もあります。

---

## 3. 支払い報告フォーム（入力内容）

請求書詳細ページで、**ステータスが「支払い待ち」（pending）のときだけ**「支払い報告」フォームが表示されます。

| 入力項目 | 必須 | 説明 |
|----------|------|------|
| **支払い日** | ✅ 必須 | 入金日（振込日）。日付入力 |
| **振込名義** | ✅ 必須 | 法人名または振込時の名義（例：株式会社◯◯薬局、山田太郎） |
| **確認用メモ** | 任意 | 控え番号など、運営が照合するときに役立つ情報 |

入力後、「報告する」をクリックして送信します。

---

## 4. 送信時の動作（システム側）

1. **API**  
   `POST /api/payments/:id/report` に  
   `pharmacyId`, `paymentDate`, `transferName`, `confirmationNote` を送信
2. **バリデーション**  
   - 対象の支払いが「pending」であること  
   - 薬局IDが一致していること  
   をチェック
3. **DB更新**  
   - `payments` テーブルを更新  
     - `paymentStatus` → **`reported`**（支払い報告済み）  
     - `paymentDate`（入金日）  
     - `transferName`（振込名義）  
     - `confirmationNote`（メモ）  
     - `reportedAt`（報告日時）
4. **運営へのメール送信（Resend）**  
   - 送信先: **info@yaku-navi.com**（環境変数 `PAYMENT_REPORT_NOTIFY_EMAIL` で変更可能）  
   - 件名: 【薬ナビ】支払い報告がありました（薬局名 / 請求書#ID）  
   - 本文: 請求書ID・契約ID・薬局名・入金日・振込名義・確認用メモ・管理画面へのリンク  
   - `RESEND_API_KEY` が未設定の場合は送信スキップ（報告処理は成功のまま）
5. **画面**  
   薬局には「支払い報告を送信しました。運営の確認をお待ちください。」と表示され、**請求書一覧**（`/pharmacy/payments`）に戻る

---

## 5. 運営（管理者）側：入金確認

| 項目 | 内容 |
|------|------|
| **画面** | 管理画面 → **支払い管理**（`/admin/payments`） |
| **フィルター** | 「報告済み（確認待ち）」で、薬局が報告した支払いを一覧表示 |
| **表示内容** | 請求書番号・契約情報・**入金日（支払い日）**・**振込名義（法人名等）**・報告日時・メモ など |
| **操作** | 実際の入金を確認し、該当支払いの「支払いを確認」を実行 |

「支払いを確認」で `POST /api/payments/:id/confirm` が呼ばれます。

---

## 6. 管理者が「支払いを確認」した後の処理

1. **payments テーブル**  
   - `paymentStatus` → **`confirmed`**  
   - `confirmedAt` を記録
2. **contracts テーブル**  
   - 該当契約の `status` → **`active`**（契約成立）  
   - `paymentConfirmedAt` を記録
3. **通知**  
   - **薬剤師**に「契約が成立しました。薬局の連絡先が開示されます」といった通知（メール・システム通知）
   - （実装次第で薬局にも「支払いが確認されました」などの通知）

---

## 7. 確認後の薬局・薬剤師の状態

- **薬局**  
  - 契約詳細で **薬剤師の連絡先**（氏名・電話番号・メールアドレスなど）が表示される  
  - 勤務スケジュールや初回出勤日を直接やり取りできる
- **薬剤師**  
  - 契約詳細で **薬局の連絡先**が表示される  
  - 同様に直接連絡可能

---

## 8. 流れのまとめ（時系列）

```
薬局が手数料を振り込む
    ↓
薬局が「請求書・支払い管理」から該当請求書の詳細を開く
    ↓
「支払い報告」フォームで
  ・入金日（支払い日）
  ・振込名義（法人名等）
  ・確認用メモ（任意）
を入力して「報告する」
    ↓
payments.status = reported、入金日・振込名義・reportedAt を保存
    ↓
運営が管理画面の「支払い管理」で報告一覧を確認
    ↓
入金を確認し「支払いを確認」を実行
    ↓
payments.status = confirmed、contracts.status = active
薬剤師へ契約成立・連絡先開示の通知
    ↓
薬局・薬剤師双方で連絡先が表示され、直接連絡可能になる
```
